## How the BA agent uses this

1. **Section 1 of interrogation** — designer mentions a maths concept and age group
2. BA reads `outcomes-index.md` to identify the outcome code(s) — e.g. `MA2-GM-03` for Stage 2 angles
3. BA reads the relevant stage file (e.g. `stage-2.md`) to confirm:
   - Exact outcome description
   - Content detail (what students specifically learn)
   - Teaching advice (how teachers approach it — informs game mechanic suggestions)
4. BA proposes level structure mapped to outcomes — e.g. L1 = Stage 2, L2 = Stage 3
5. Outcome codes go into README, deployment manifest, and report email copy

| Prefix | Meaning |
|--------|---------|
| `MAE-` | Early Stage 1 |
